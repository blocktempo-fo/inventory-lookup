期間：2026/04/13 – 05/08（約 3.5 週）  |   負責人：Kessy（@cheunghy305）

一、試行期概況

  • 已登記 115 件器材、12 套組（涵蓋 60+ 件配件關聯）
  • 功能運作正常：查詢 / 篩選 / 借用 / 歸還 / 套組推薦 / 替代品 / 批次處理
  • 手機與電腦雙端皆可使用，跨分頁即時同步

二、收到的回饋與已改進事項

回饋原因分析已採取改進
借用 / 歸還流程載入太慢 批次借用為序列發送 API（6 件 ≈ 18 秒）；UI 凍結等回應 改為 Promise.all 平行發送 + 樂觀更新（UI 即時反映）+ spinner 動畫；體感從 18s 降至 < 1s

三、試行期觀察到的根本問題

除了上述具體回饋，試行期最關鍵的發現是：仍有人不填表就直接拿器材。

這不是工具的 bug，而是行為層面：

原因解法方向
急用，直接拿走改變取用動線
「等等再補填」，後來忘記讓登記發生在取用之前
沒有被擋下的經驗不填就拿不到

填寫成本已壓到最低（30 秒可完成、套組一鍵借），剩下能改的是取用流程本身。

四、下一階段：體貼式流程

借用 = 服務，不是手續。Kessy 從「攔截者」變「禮賓」——填單後幫你準備好、直接拿給同仁，不填就拿不到。

新流程

  1. 同事填借用單（約 30 秒）
  2. Kessy 即時收到 Telegram 通知
  3. Kessy 取出器材、檢查、直接拿到同仁座位 / 約定點交付
  4. 完成

實作方式

動作負責
